前言

研究了許久，小弟終於發現為何各位30CM、E CUP 的攻城獅大大們要寫自己的個人的網站呢？全部都因為走在時尚尖端嗎? 還是為了有一個地方能『 夠分享生活中的觀察 』以及 『 記錄個人所學及經歷 』！。所以身為一個專業的攻城獅壹定要有一個潮到出水的部落格才行！

使用 Github Page 搭建 Hexo

Step 1 工欲善其事，必先利其器

以普通的情況來說，必須要先有一個架站空間！但你自己擁有一個架站空間可能會有什麼煩人的事情要處理呢？

  1. 必須要先租一個主機空間，你必須要維護。
  2. 維護不是一件困難，困擾的是你除了寫文章之外還需要碰一些技術

如果你有一次忘記繳費，或者一些其他的原因導致辛辛苦苦建立的部落格暫時斷線，或者忘記繳錢被刪除資料是不是很氣餒呢？
所以我們需要有一個空間不需要讓我們擔心一堆煩人的問題，只要好好的當一個寫手 ，我的意思是專業的攻城獅，紀錄就好了，所以為了讓我們可以專心寫部落格，我們可以將網站放在最令人安心的 GitHub Page 開始！

建立 Github Page 環境

Step 1 註冊一個 Github 帳號

怎麼註冊在這邊就不再贅述 請右轉 GitHub

Step 2 搞定 GitHub Page !

靜態網頁好夥伴 GitHub Pages ！ 它是可以給用戶編寫、託管在 Github 上的靜態網頁 。在申請玩 Github 帳號之後，可以看正中間有一個大大的 Start a Project 給他大力按下去就對了！
如圖：



將專案名稱取名為 username.github.io 並按下最下方的 Create repository
P.S 本文章 yourname 或 username 指的就是你的 Github 帳號唷！

恭喜你建立好 Github Pages 的專案了！
現在就馬上點進 http://username.github.io ，你會發現 有東西 才怪！
哦原來是我還沒把東西放上來呀！沒錯 Github Pages的預先設置就是這麼簡單 ，接下來要部署 Hexo上來了！部署完之後才會有東西唷！

建立 Hexo 環境

這裏有一個前提，我是用 Mac 電腦，如果你用的是 Linux 也大同小異 ，但是如果你是 微軟 的話，出口在直走右轉唷！關鍵字是 windows 安装 hexo，好接下來開始吧！

Hexo 需要安裝以下系統環境來支援：

  1. Node.js
  2. Git
  3. Xcode ( Mac用戶必須安裝 )

安裝好了之後我們可以開啟我們的終端機，並輸入以下指令安裝 Hexo：

$ npm install -g hexo-cli

安裝完之後我們便可以來建立網站了！

$ hexo init yourname ( 初始化新的 Hexo )
$ cd yourname  ( 進入您剛剛建立的 Hexo 資料夾當中 )
$ npm install （ 安裝 Hexo )

yourname 就是指在電腦裡的檔案名稱，可以隨意取，例如『 myhexoblog 』這樣就完成安裝了！

設定 Hexo

接下來就要開始設定 Hexo 讓你的網站可以個人化，符合建立自己網站的宗旨唷！

進入您 Hexo 主目錄下之後，先找到_config.yml 這個檔案！

title: 不為繁華易匠心 (輸入您的標題)
subtitle: 匠心，是傾注於作品之中的精神、情感、乃至魂魄。 （輸入您的至理名言）
description:(輸入您的網站描述)
author: 王性驊 （輸入您的姓名）
language: zh-TW （輸入您所使用的語言）
timezone: (留空可以使用系統時間！)

這樣就第一步完成您的設定了！
接下來，要找到上傳至 github 的設定，使用「搜尋」找出 deploy:

deploy:
   type: git
   repo: https://github.com/username/username.github.io.git
   branch: master
username 就是指你自己的github帳號，記得改對，然後按存檔。

然後回到你的終端機，先進入 yourname 資料夾
並依序輸入以下指令：

$ hexo clean (清除快取)
$ hexo g （建立靜態檔案）
$ hexo d （在 Github Page 上顯示）

這三個指令就是幫我們把網站自動生成並發佈到 http://username.github.io 上的指令，完成後應該就可以打開瀏覽器看看自己網站發布成功的樣子囉！
