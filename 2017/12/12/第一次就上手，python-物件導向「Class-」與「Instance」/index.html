前言

在解釋前一篇所敘述的物件導向程式設計的三種特性「 封裝 」、「 繼承 」、「 多型 」之前，除了必須要對物件導向有一些基礎的瞭解之前，還必須知道物件導向最重要的兩個概念，那就是 Class 與 Instance

類別與實例

Question : 為什麼要使用物件導向程式設計呢？

Ans : 物件導向，在思考問題的時候，其實是想把抽象的問題建立一套標準的模板，這樣讓設計軟體的層次清楚，有使用到什麼模組一目瞭然，方便維護與理解。然而在這樣的設計當中，類別 與 實例是特別重要的概念。

舉例說明，前一篇提到的範例 People 這個類別（ Class ），就是一個抽象的模板，而實例(Instance)
就是根據這個模板創造出來的資訊，像是王先生、蔡小姐等具體的對象，每個對象擁有相同的方法，但是各自所擁有的資料可能不同。
前一篇連結

這次再深入一些，我們用動物來舉個例子吧！ 在程式設計師的眼中，動物要怎麼變成一個模板，與把它實例化呢？
在 Python3 當中，想要定義類別，是透過一個關鍵字 Class 來宣告：

class Animal(object):
    pass

Class 後面接著的是 class 的名稱再來就是他有沒有繼承什麼其他的類別（類別之後會提到），最後下面是這個類別做的事情了。
定義好類別之後，這就是我們所謂的抽象方法，接下來會依據這個類別創建出實例，如果您定義好整個類別後，將他實例化，您就可以看到他的差別。

# 實例化 Animal 這個類別，將它定義成狗
Dog = Animal()

您可以印出來看看，如果已經
如果是印出 Dog 的話，您應該會看到像這樣的一串字， <__main__.animal object="" at="" 0x109a67a90="">
意思是 Dog 的實例已經被指派到記憶體空間 0x109a67a90 當中。
如果你印出 Animal 這個 Class 的話 應該會看到這個 ，這就沒有被指派到記憶體空間當中了，
每次讓 Class 實例化到記憶體空間當中的位置都是不一樣的。

知道了 Class 與 Instance 之後呢，我們在用動物舉一次例子吧！
讓我們用物件導向的概念將動物給模板化

class Animal(object):

    """
    宣告一個抽象的類別 Animal 
    """
    # __init__是 python 當中的一個特殊方法，可以在一開始時，就將這個累所需要的屬性綁定，我們稱之為初始化
    def __init__(self, age, sound, 姓名):
        """
        假設動物有三種屬性，年齡，叫聲，姓名，當然你也可以舉更多屬性
        """
        self.age = age
        self.sound = sound
        self.__name = name

    # 初始化屬性之後，動物可能會有一些動作，我們可以把它看成物件導向當中的方法（ Method ）
    def run(self):
        '''動物跑步的方法'''
        pass

    def Animal_sounds(self):
        '''動物叫的方法'''
        print(self.sound)

注意：由於類別可以起到模板的作用，因此創建實例的時候，我們可以把一些我們認為必須的屬輕強制填寫進去，透過一個特殊的方法init。

注意：init 前後分別有兩個下劃線，在類別當中，第一個參數永遠是 self，表示創建的實例本身，因此在 init 方法內部，可以將每一個屬性都綁定到 self 當中，這樣就可以在這個實例內自由地使用了！

實例化 Class 的方法

# 這樣我們就創造了一隻 叫做 jj 的狗 ，年齡是 18 歲，叫的聲音是 wof
Dog = Animal('18', 'wof', 'jj')

注意：你一定很好奇，上面再創造類別時有傳入 Self 參數，但我在實例化時為何沒有填入呢？因為 python 很聰明自己會幫我們把它傳進去，所以就不用填拉

物件導向第一個特性：封裝

再一次強調了一些物件導向的基本之後，終於要講到第一個特性，封裝了，我非常期待呢！

物件導向的一個重要的特點就是封裝。在上面 Animal 的這個類別當中，每個實例就擁有自己的 Name 、 sound 、 age 這些資訊，我們可以通過一些函數來呼叫這些資訊，舉例來說，想要聽到這個動物的叫聲:

    print (Dog.sound)

這樣就是通過外部的方法，取得 Dog.sound 的值，你必須明確的知道，印印出來需要使用print 的這個指令，但是透過封裝，只需要透過這個 class 的方法，就可以自動地幫我們印出這隻狗的叫聲了:

Dog.Animal_sounds('wof')

在本例子當中，因為 Animal_sounds()這個方法當中也只有一行指令，如果裏面包含了一千行，你在外面只需要用一行就可以執行一千行的功能，這就是封裝，是不是很方便呢?

封裝的另外一個好處，就是可以給 Animal 這個 Class 增加新的方法，例如:狗會跳!
只要在完整的 class 下方新增一個跳躍的方法就好了，不用做其他的更動

class Animal(object):
    .....

    def jump(self):
        """狗跳耀的方法"""
        print ("Jump!")

這樣就完成了狗跳耀的方法了!

小結

Class 就是一種創建實例的模板，而實例就是一個一個具體的物件，每一個實例傭有的資訊都互相獨立，互不影響。

方法就是與實例綁定的函數，跟普通函數不一樣的是，方法可以直接呼叫實例的資訊，但是不用知道具體是怎麼實現的。
