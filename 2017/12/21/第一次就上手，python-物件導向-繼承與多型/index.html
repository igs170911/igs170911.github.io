Python 的繼承與多型

前言

這是一個相當好用的功能，必學，由於 Python 沒有抽象類別，晚點也會順便介紹 Python 的抽象類別！

Python 的繼承

簡單的來說，在物件導向程式設計當中，當我們定義一個 Class Ａ的時候，可以從某可現有的 Class Ｂ當中，使用該的功能而不用重新做，也就是說擁有該類別的某些屬性，以及方法的話，我們就可以稱為 A 繼承 B。 A 是 B 的子類別， 而被繼承的 Ｂ 則稱為 基類別、父類別或是超類別（Base Class 、Super Class）。如果不了解屬性以及方法請參閱 第一次就上手，Python 物件導向簡介篇

舉例說明，前面的篇幅已經寫了一個名為 Animal 的 Class ，而且有一個方法 run()

class Animal(object):
    .....
    def run():
        print ("Animal is running")

當我們需要撰寫一隻貓類別以及一隻狗的類別的時候，我們可以直接繼承 Animal 這個類別例如：

class Dog(Animal):
    pass

class Cat(Animal):
    pass

對於 Dog 來說， Animal 就是他的父類別
對於 Animal 來說，Dog 就是他的子類別
繼承有什麼好處呢？
最大的好處就是子類別直接獲得了父類別所有的功能
舉例說明，Animal 實作了一個 run() 功能，那麼 Dog 裡面就算只有 pass ，由於繼承的關係
他直接擁有 run()這個方法！我們直接跑看看吧！

dog  = Dog()
dog.run()

你印出來的結果應該為

Animal is running...

繼承除了剛剛所說的好處之外，也可有另外一個好處，那就是改寫，什麼是改寫呢？

class Dog(Animal):

    def run(self):
        print('Dog is running...')

    def jump(self):
        print("Jump!")

也就是說如果子類別有 run() 方法的話，子類別的run()將會取代父類別的run() 方法
你跑

dog = Dog()
Dog.run()

你得到的結果將會是 

Dog is running...

這樣也就是繼承的第二個好處，我們也就稱為多型。

Python 的多型

想要了解多型的話，我們必須要先了解一下 Python 的資料型態。當我們定義一個 class 的時候，實際上我們就定義了一種資料型態。我們定義的資料型態跟 Python 預設擁有的資料型態，例如 str、list、dict 其實是都一樣的！

# a 是 list() 型態
a = list()
# b 是 Animal() 型態
b= animal()
# c 是 Dog() 型態
c = Dog()

我們用來判斷一個變數是不是某種資料型態可以利用 isinstance()來做判斷

isinstance(a, list)
True

isinstance(b, Animal)
True

isinstance(c, Dog)
True

執行完後就可以發現對應的類型確實都是正確的
但是如果你測試

isinstance(c, Animal)
True

的話， c 因為繼承的關係，他同時也是 Animal 這個類行的資料型態喔！
但是反過來的話呢？

b = Animal()
isinstance(b, Dog)
False

反過來就不行了，舉例說明因為 Dog 可能有父類沒有的方法所以無法到推回去，就算沒有也不行！
了解了基本的資料型態之後，我們寫一個簡單的範例看看多型吧！

我們撰寫一個簡單的 Function 來測試看看

def runTenTime(animal):
    for i in range(0,10):
        animal.run()

這是一個簡單的跑十次的 Function 當你傳進來的 class 有 run() 方法的時候就可以正常執行
當我們傳入的 class 為 Animal 的時候，跑出來的結果為

當我們傳入的 class 為 Dog 的時候，跑出來的結果為

看到這邊你心中會不會有一個疑問
咦，我這不是在寫廢話嗎？這不是本來就這樣嗎？有什麼特別的呢？
但是如果我再多寫一個繼承 Animal 的類別呢？

class bird(Animal):
    def run():
        print ("bird is flying!")

你在跑看看

runTenTime(bird())

你得到的結果是

bird is flying!
bird is flying!

你有發現什麼神奇的事情嗎？
有，如果你只要新增一個繼承 Animal 的子類別，不用對 runTenTime做任何修改，只要是任何繼承 Animal作為參數的函數或者是方法，都可以不用修改，正常的運作，原因就在於多型，雖然每個子類別的 Run 方法都不一樣，我們視為不同的資料型態，但是只要各自實作run的話都可以正常的運作唷！這就是多型的意思
換句話說
只要是對於同樣的一個 Class ，我們只需要知道他的類型是什麼，無需確切的知道他的其他子類型，就可以放心地使用父類型當中所提供的方法，經由運作時的對象才確切地決定要用哪一種資料型態下去實作，這就是多型真正的威力，呼叫的時候只管呼叫而不用去管理細節，當我們新增一種 Animal 的子類別的時候，只要確定 run() 有被正確的寫出來，而不用去管理原來的程式是如何呼叫的，就可以了。

貼心小提示：上面所講的其實也就是物件導向程式設計中的開放封閉原則的例子

對擴展開放：允許新增 Animal 的子類別
對修改封閉：不需修改依賴 Animal 形態的 runTenTime()函數（現在扯遠了，以後會提！）

貼心小提示：
對於動態語言與靜態語言來說，多型的規定有著程度上面的不同，對於靜態語言來說，以 Java 為例子我們想要使用 runTenTime 時所傳入的 Class 必須是 Animal 或者是他的子類別才可以，否則根本無法呼叫 run() 這個方法。
而對於動態語言來說規定則沒有這麼嚴格，以 Python 為例，他其實並不嚴格要求要繼承 Animal ，只要所傳入的東西有 run() 方法就可以直接執行了，這也是動態語言著名的鴨子型別。一個物件只要看起來像鴨子，走路也類似鴨子，那他就可以是一隻鴨子！

結語

繼承，可以把富類別的功能都拿過來直接做使用，輪子不必從頭做起，子類別可以新增自己特有的方法，也可以把富類別不適合自己的方法給重寫，是非常方便的功能呢！
